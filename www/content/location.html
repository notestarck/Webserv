Tester les locations

Qu'est ce qu'une location ?

Une "location" dans un fichier de configuration Nginx est un bloc de directives qui commence par la directive "location" suivie d'un motif d'URL. Le motif d'URL peut être une chaîne de caractères littérale ou une expression régulière qui correspond à une ou plusieurs URL spécifiques.

Voici un exemple de bloc "location" dans un fichier de configuration nginx : 

 location /images/ {
    root /content;
    allow GET;
    index portfolio.html";
 }

  2. Pour le second test, 'location /page2' a été defini à deux reprises dans le fichier nginx. Uniquement le deuxième doit être pris en compte. Vous devriez arriver sur la page de documentation du projet.

  3. Pour ce trosieme test, la 'location /page3' n'a pas la bonne methode, vous devez recevoir le code erreur 405 Method Not Allowed.

  4. Pour ce quatrieme test, la 'location /Aindex' doit permettre d'archiver le contenu du dossier 'path'.

  5. Pour ce dernier test, nous allons tester les redirections. Dans le fichier de '.conf', une redirection a ete mise en place : 'location /redir'. En rajoutant ceci a l'url, vous serez rediriges, vers la page index.
